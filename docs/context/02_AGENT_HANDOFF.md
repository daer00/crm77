# Правила передачи контекста между чатами

## Как работать каждому следующему чату
1. Сначала прочитать:
   - `docs/context/01_PRODUCT_TZ.md`
   - `docs/context/03_AGENT_LOG.md`
   - `docs/context/04_STAGE_CHECKLIST.md`
2. Зафиксировать, на каком этапе работа.
3. Не ломать уже реализованные фичи без явной причины.
4. Перед завершением обновить `03_AGENT_LOG.md`.

## Формат записи в лог
```md
## YYYY-MM-DD HH:MM | Этап
- Что сделано:
- Что изменено (файлы):
- Ограничения/риски:
- Что должен сделать следующий агент:
```

## Инварианты (обязательны для всех чатов)
- Никакой работы с `.env` (чтение/редактирование запрещено).
- Минимум лишнего кода: решения должны быть простыми и короткими.
- Все этапы фиксируются в git-коммитах.
- Перед новым этапом проверять, что приложение запускается.
- Изменения по UI не должны ломать мобильную верстку.

## Приоритеты
1. Корректность данных и бизнес-логики.
2. Безопасность (авторизация, роли, подтверждение удаления).
3. Производительность и UX.
4. Визуальные улучшения.
